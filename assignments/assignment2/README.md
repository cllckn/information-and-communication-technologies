## Objective
The goal of this assignment is to design and develop a **web-based e-commerce service** using **Node.js** and either **PostgreSQL** or **MongoDB**.  

This assignment emphasizes **web service design**, **API development**, and **database integration** using modern backend technologies.

## Tasks/Requirements

The system represents an **E-Commerce platform** that supports:
- Managing customers and employees
- Managing products and inventory
- Adding and viewing orders
- Placing new orders linked to specific customers and products

The backend service should:
- Store and retrieve data using PostgreSQL or MongoDB
- Handle requests and responses in **JSON format**
- Use appropriate **HTTP methods** and **status codes**


1. Develop a **web service** using **Node.js**.
2. Use one of the following databases:
   - **PostgreSQL** 
   - **MongoDB** 
3. Implement the following operations (services):
- **Customer**: Add, update, list, view, and delete customer records.
- **Employee**: Manage employees and their roles.
- **Product**: Add new products, update inventory, list products, and remove them.
- **Order**: Add new orders, view order details, and calculate totals.
- **Placing an Order**: Associate orders with customers and products.

4. Provide  **simple interfaces (frontend)** for testing the services.
5. Perform REST API testing using any of the following:
   - HTTP client
   - curl
   - Postman
   - Equivalent testing tool
6. Include a **UML Deployment Diagram** and a **Database ER Diagram** (or Document Model for MongoDB).

7. **Documentation**
    - Provide a **report** containing:
        - Requirement analysis
        - UML deployment diagram
        - Design and Implementation notes
        - Include the Git repository address or the source code as an annex to the report.

8. **Presentation & Oral Exam**
    - Run your application in front of the instructor.
    - Present and explain your report.
    - Defend your design and code.
